# ADR 0001: Bounded Contexts and Ubiquitous Language

## Status
Proposed

## Context
To transition the "Zero Sum RPG" engine to a AAA state-of-the-art cyber-warfare simulation, we must enforce strict Domain-Driven Design (DDD) via Hexagonal Architecture. The core logic must be completely isolated from infrastructure concerns (e.g., Firebase RTDB). 

This document defines the system boundaries and Ubiquitous Language for our two most critical domains: the `ZeroSumLedger` and `Psychophysics`.

---

## 1. Bounded Context: ZeroSumLedger
**Responsibility:** Enforces the brutal zero-sum economic transfers of the simulation. It ensures that life support mechanics obey the law of equivalent exchange—no character can be healed without an irreversible, logged casualty elsewhere in the facility.

### Ubiquitous Language
*   **Ledger:** The immutable append-only record of all zero-sum transactions (trauma events).
*   **Equivalent Exchange:** The core domain rule stating that `Player Healing Amount` must precisely equal the `Civilian Life Support Drain`.
*   **Emergency Heal:** A specific Command issued by a Player to restore Hit Points (HP).
*   **Civilian Entity:** A non-player character represented mathematically as a life-support resource constraint within the facility.
*   **Trauma Event:** A Domain Event emitted when an Emergency Heal forces a Civilian Entity to become a casualty.
*   **Casualty Log:** A read-model projection of Trauma Events used by the GM and Spectator views.

---

## 2. Bounded Context: Psychophysics
**Responsibility:** Simulates acoustic physics for stealth tracking and physiological strain via Allostatic Load models. This context handles the chaotic interface between the physical world (dB levels, Android hardware tremors) and the digital game state.

### Ubiquitous Language
*   **Allostatic Load (Stress):** A quantifiable value object (0-100) representing a character's cumulative psychological and physical strain.
*   **Cyberpsychosis Threshold:** A strict domain rule constraint. When `Allostatic Load` crosses 75, the state transitions to `Psychotic`.
*   **Acoustic Emission:** A transient event containing a `dB` value representing noise generated by an action (e.g., gunshot, footsteps).
*   **Signal-to-Noise Ratio (SNR):** A dynamically calculated value object derived from the Inverse Square Law comparing an `Acoustic Emission` against ambient environmental noise.
*   **Detection Radius:** The geographical distance at which an `Acoustic Emission` SNR breaches the threshold of observability.
*   **Haptic Feedback Signal:** A Domain Event triggered by crossing the Cyberpsychosis Threshold, commanding the infrastructure layer to initiate physical device tremors.
